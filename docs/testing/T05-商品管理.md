## 二、商品状态流转

| # | 场景 | 操作步骤 | 预期结果 | 通过 |
|---|------|----------|----------|------|
| 1 | 草稿 → 仓库 | 草稿商品 → 保存并放入仓库 | 状态变为 WAREHOUSE | |
| 2 | 仓库 → 上架 | 仓库商品 → 点击上架 | 状态变为 ON_SALE，小程序可见 | |
| 3 | 上架 → 下架 | 已上架商品 → 点击下架 | 状态变为 OFF_SALE，小程序不再展示 | |
| 4 | 下架 → 重新上架 | 已下架商品 → 点击上架 | 状态变为 ON_SALE，小程序重新可见 | |
| 5 | 草稿直接上架（非法） | 尝试从草稿直接上架 | 应被拒绝，必须先放入仓库 | |
| 6 | 下架后历史订单 | 商品下架 | 已有订单数据保留，不受影响 | |

## 三、规格与SKU

| # | 场景 | 操作步骤 | 预期结果 | 通过 |
|---|------|----------|----------|------|
| 1 | 无规格商品 | 创建商品不添加规格组 | 系统自动生成一个默认 SKU | |

| 6 | 一个商品最多一个主规格组 | 尝试设置两个规格组都为主规格组 | 应被拒绝 | |
| 7 | 统一积分 | 同一商品的不同 SKU | 所需积分一致，由 SPU 统一维护 | |

## 五、限购规则
