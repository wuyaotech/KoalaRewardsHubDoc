# 测试：M13-定时任务

> 对应需求：[M13-定时任务](../modules/M13-定时任务.md)

## 一、超时订单自动取消

| # | 场景 | 前置条件 | 预期结果 | 通过 |
|---|------|----------|----------|------|
| 1 | 正常超时取消 | 订单 PENDING_VERIFY，核销码已过期 | 订单变 CANCELLED，积分退回（IN 流水），库存恢复，收到 ORDER_CANCELLED_TIMEOUT 消息 | |
| 2 | 未超时不处理 | 订单 PENDING_VERIFY，核销码未过期 | 不处理 | |
| 3 | 已核销不处理 | 订单 VERIFIED | 跳过 | |
| 4 | 幂等重跑 | 重复调用 /cron/daily | 已取消的订单不重复处理，积分不重复退 | |
| 5 | 退积分准确 | 订单消耗大RF 100 + 小RF 50 | 大RF 退 100，小RF 退 50，余额正确 | |

## 二、注册周年积分清零

| # | 场景 | 前置条件 | 预期结果 | 通过 |
|---|------|----------|----------|------|
| 1 | 正常清零 | 会员 activatedAt 周年日 = 今天，余额 > 0 | 大RF + 小RF 全部清零，每个账户一条 EXPIRE 流水，收到 POINTS_EXPIRED 消息 | |
| 2 | 余额为 0 不处理 | 周年日到达但余额已为 0 | 不写流水，不发消息 | |
| 3 | 非周年日不处理 | 不是今天的周年日 | 跳过 | |
| 4 | 未满一年不处理 | activatedAt 距今不到一年 | 跳过 | |
| 5 | 清零后等级不降 | 清零前 Lv.3 | 清零后等级仍为 Lv.3 | |
| 6 | 幂等重跑 | 同一天重复调用 | 检查已有 EXPIRE 流水则跳过 | |
| 7 | 含多个小朋友 | 会员有 2 个小朋友各有小RF积分 | 每个小朋友的小RF账户都清零 | |

## 七、整体执行

| # | 场景 | 操作步骤 | 预期结果 | 通过 |
|---|------|----------|----------|------|
| 1 | 正常执行 | 调 POST /cron/daily | 返回所有子任务的 processed/skipped/failed 统计 | |
| 2 | 部分失败 | 某条记录处理异常 | 该条失败不影响其他记录，errors 数组记录失败详情 | |
| 3 | 空跑 | 无任何待处理数据 | 正常返回，各子任务 processed = 0 | |
| 4 | 执行耗时 | 正常数据量 | 30 秒内完成 | |
