# 测试：M09-消息通知

> 对应需求：[M09-消息通知](../modules/M09-消息通知.md)

## 一、消息自动生成

| # | 场景 | 触发操作 | 预期结果 | 通过 |
|---|------|----------|----------|------|
| 1 | 兑换成功消息 | 用户兑换商品成功 | 自动生成一条消息："您已成功兑换「商品名称」，请前往核销" | |
| 2 | 核销完成消息 | 工作人员核销订单 | 自动生成一条消息："您兑换的「商品名称」已核销完成" | |
| 3 | 积分清零提醒（3个月前） | 距清零日 3 个月 | 生成提醒消息（当前未实现） | |
| 4 | 积分清零提醒（2个月前） | 距清零日 2 个月 | 生成提醒消息（当前未实现） | |
| 5 | 积分清零提醒（1个月前） | 距清零日 1 个月 | 生成提醒消息（当前未实现） | |
| 6 | 不重复生成 | 同一事件重复触发 | 不产生重复消息 | |

## 二、消息红点

| # | 场景 | 操作步骤 | 预期结果 | 通过 |
|---|------|----------|----------|------|
| 1 | 有未读消息时 | 生成新消息后 → 查看消息入口 | 消息中心入口显示红点 | |
| 2 | 全部已读后 | 所有消息都已读 | 红点消失 | |
| 3 | 新消息到达 | 已全部已读 → 新消息生成 | 红点重新出现 | |

---

## 三、消息列表（小程序端）

| # | 场景 | 操作步骤 | 预期结果 | 通过 |
|---|------|----------|----------|------|
| 1 | 查看消息列表 | 进入消息中心 | 展示所有消息，区分已读/未读 | |
| 2 | 标记已读 | 点击一条未读消息 | 该消息变为已读状态 | |
| 3 | 空消息列表 | 无任何消息时进入消息中心 | 展示空状态提示 | |
| 4 | 消息排序 | 多条消息 | 按时间倒序排列，最新消息在前 | |
| 5 | 分页加载 | 消息较多时 | 下拉/滚动分页加载 | |
