# 测试：M02-会员信息

> 对应需求：[M02-会员信息](../modules/M02-会员信息.md)

## 一、线下开卡 / 会员建档（PC端）

| # | 场景 | 操作步骤 | 预期结果 | 通过 |
|---|------|----------|----------|------|
| 1 | 正常建档 | 后台输入手机号、姓名 → 提交 | 创建会员档案，状态为"未激活"，建档来源"线下录入" | |
| 2 | 手机号已存在 | 输入已存在的手机号建档 | 提示手机号已存在，不可重复建档 | |
| 3 | 手机号格式错误 | 输入非法手机号（如少位、多位、非数字） | 表单校验不通过 | |
| 4 | 必填字段缺失 | 不填姓名直接提交 | 提示必填字段不能为空 | |

## 二、个人信息编辑（小程序端）

| # | 场景 | 操作步骤 | 预期结果 | 通过 |
|---|------|----------|----------|------|
| 1 | 补充个人信息 | 进入个人信息页 → 填写姓名、性别、生日 → 保存 | 信息保存成功，个人中心显示更新后的信息 | |
| 2 | 修改已有信息 | 修改姓名 → 保存 | 更新成功 | |
| 3 | 头像修改 | 上传新头像 | 头像更新成功，各页面同步展示新头像 | |
| 4 | 手机号不可修改 | 尝试在小程序端修改手机号 | 手机号字段不可编辑，提示联系工作人员 | |

## 三、小朋友管理（小程序端）

| # | 场景 | 操作步骤 | 预期结果 | 通过 |
|---|------|----------|----------|------|
| 1 | 添加第一个小朋友 | 亲子绑定页 → 填写姓名 → 保存 | 添加成功，个人中心出现小RF积分区域 | |
| 2 | 添加多个小朋友 | 继续添加第 2、3 个小朋友 | 每个小朋友独立显示，各自有独立的小RF积分 | |
| 3 | 编辑小朋友信息 | 修改小朋友姓名、性别、生日 | 更新成功 | |
| 4 | 删除小朋友 | 删除一个小朋友 | 小朋友从列表移除，对应小RF积分账户软删除 | |
| 5 | 删除唯一小朋友后 | 删除最后一个小朋友 | 个人中心小RF积分区域消失，商城小RF Tab 隐藏 | |
| 6 | 未绑定小朋友时 | 未添加任何小朋友 → 查看个人中心 | 不展示小RF积分区域 | |

## 四、会员列表与详情（PC端）

| # | 场景 | 操作步骤 | 预期结果 | 通过 |
|---|------|----------|----------|------|
| 1 | 搜索会员 | 输入手机号或姓名搜索 | 返回匹配的会员列表 | |
| 2 | 筛选会员状态 | 筛选"已激活"/"未激活"/"已停用" | 列表只展示对应状态的会员 | |
| 3 | 查看会员详情 | 点击某会员进入详情页 | 展示完整画像：资料 + 积分 + 等级 + 兑换记录 + 券 | |
| 4 | 后台修改会员信息 | 修改会员姓名、手机号等 | 修改成功，保留修改记录 | |
| 5 | 后台修改手机号 | 将会员手机号从 A 改为 B | B 未被占用则修改成功；B 已存在则提示冲突 | |
| 6 | 会员导出 | 点击导出按钮 | 下载包含会员数据的文件 | |

## 五、用户类型与状态

| # | 场景 | 操作步骤 | 预期结果 | 通过 |
|---|------|----------|----------|------|
| 1 | 未绑定手机号用户 | 仅微信授权未绑定手机号 | 可登录小程序，但非正式会员，不能兑换 | |
| 2 | 未激活会员 | 后台建档但用户未登录小程序 | 后台可见该会员，可录入积分，但会员未绑定登录账号 | |
| 3 | 已激活会员 | 绑定手机号或线下建档后完成绑定 | 完整会员功能，可兑换、查看积分等 | |
