# 测试：M06-兑换交易（含核销）

> 对应需求：[M06-兑换交易](../modules/M06-兑换交易.md)

## 一、心愿单

| # | 场景 | 操作步骤 | 预期结果 | 通过 |
|---|------|----------|----------|------|
| 1 | 添加心愿单 | 商品详情 → 加入心愿单 | 收藏成功，心愿单列表可见 | |
| 2 | 移除心愿单 | 心愿单页 → 移除 | 移除成功 | |

## 二、购物车

| # | 场景 | 操作步骤 | 预期结果 | 通过 |
|---|------|----------|----------|------|
| 1 | 加入购物车 | 商品详情 → 选择规格 → 加入购物车 | 购物车中出现该商品 | |

| 3 | 大RF/小RF分组展示 | 购物车同时有大RF和小RF商品 | 分组展示 | |
| 4 | 小RF商品关联小朋友 | 加入小RF商品到购物车 | 需选择关联到哪个小朋友 | |
| 5 | 修改购物车数量 | 增加/减少商品数量 | 数量变更正确，积分小计更新 | |
| 6 | 从购物车移除 | 删除购物车中的商品 | 移除成功 | |

## 三、兑换下单

### 3.1 兑换预览

| # | 场景 | 操作步骤 | 预期结果 | 通过 |
|---|------|----------|----------|------|
| 1 | 正常预览 | 购物车勾选商品 → 点击结算 | 展示兑换预览：商品明细、积分小计、各类型积分消耗 | |
| 2 | 库存不足预览 | 商品加入购物车后库存被他人兑完 → 预览 | 提示库存不足 | |

### 3.2 确认兑换

| # | 场景 | 操作步骤 | 预期结果 | 通过 |
|---|------|----------|----------|------|
| 1 | 纯大RF兑换 | 只选大RF商品 → 确认兑换 | 大RF积分扣减，库存扣减，生成订单+核销码 | |
| 2 | 纯小RF兑换 | 只选小RF商品（关联小朋友A） → 确认兑换 | 仅扣减 A 的小RF积分 | |
| 3 | 混合兑换 | 同时选大RF+小RF商品 → 确认 | 大RF/小RF分别扣减，生成同一订单 | |
| 4 | 积分余额恰好等于价格 | 余额 100，兑换 100 积分商品 | 兑换成功，余额变为 0 | |
| 5 | 积分不足 | 余额 80，兑换 100 积分商品 | 兑换失败，提示余额不足 | |
| 6 | 库存仅剩 1 件 | 库存 1 → 兑换 1 件 | 兑换成功，库存变为 0 | |
| 7 | 库存为 0 | 库存已为 0 → 尝试兑换 | 兑换失败，提示库存不足 | |
| 8 | 多小朋友不可合并小RF | 小朋友A有30分、B有30分 → 兑换50分小RF商品 | 兑换失败，不可合并不同小朋友积分 | |
| 9 | 事务原子性 | 下单过程中积分扣减成功但库存扣减失败 | 整个事务回滚，积分退回 | |
| 10 | 兑换后购物车清除 | 兑换成功 | 购物车中已结算的商品被清除 | |
| 11 | 兑换后生成核销码 | 兑换成功 | 生成 8 位数字核销码，有效期 10 天 | |

## 四、核销流程

### 4.1 核销码展示（小程序端）

| # | 场景 | 操作步骤 | 预期结果 | 通过 |
|---|------|----------|----------|------|
| 1 | 查看核销码 | 兑换记录 → 点击待核销订单 | 展示二维码 + 8位数字码 | |
| 2 | 核销码有效期 | 查看核销码页面 | 展示有效期截止时间（下单后+10天） | |

### 4.2 扫码/输码核销（PC端）

| # | 场景 | 操作步骤 | 预期结果 | 通过 |
|---|------|----------|----------|------|
| 1 | 扫码预览 | 扫描用户二维码 | 展示核销预览信息：会员、商品明细、积分 | |
| 2 | 输码预览 | 手动输入 8 位数字码 | 展示同样的核销预览信息 | |
| 3 | 确认核销 | 预览后确认核销 | 状态变为 VERIFIED，记录核销时间和操作人 | |
| 4 | 核销码错误 | 输入不存在的核销码 | 提示核销码无效 | |
| 5 | 核销已完成的订单 | 对已核销订单再次核销 | 提示该订单已核销 | |
| 6 | 核销已取消的订单 | 对已取消订单尝试核销 | 提示订单已取消，不可核销 | |
| 7 | 核销过期订单 | 核销码已过期的订单 | 提示核销码已过期 | |
| 8 | 核销后双端同步 | PC端核销完成 | 小程序端该订单状态同步变为"已完成" | |

## 五、订单取消与退还

| # | 场景 | 操作步骤 | 预期结果 | 通过 |
|---|------|----------|----------|------|
| 1 | 用户主动取消 | 待核销订单 → 用户点击取消 | 状态变为 CANCELLED，积分退回，库存恢复 | |
| 2 | 取消后积分退回 | 取消后查看积分余额 | 余额恢复到兑换前的值，生成 IN 流水 | |
| 3 | 取消后库存恢复 | 取消后查看商品库存 | 库存恢复 | |
| 4 | 已核销不可取消 | 已核销订单尝试取消 | 不允许取消 | |
| 5 | 超时自动取消 | 待核销超过 10 天 | 定时任务自动取消，退积分+恢复库存（当前未实现） | |
| 6 | 超时取消幂等 | 对同一过期订单重跑取消逻辑 | 不重复处理 | |
