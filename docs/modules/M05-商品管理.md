# 模块05：商品管理

**职责**：积分商城中可兑换商品的全生命周期管理

## 能力清单

| # | 能力 | 端 | 描述 |
|---|------|---|------|
| 1 | 商品浏览 | 小程序 | 按积分类型（大RF/小RF）分别展示商品列表，支持分类筛选、积分排序 |
| 2 | 商品详情 | 小程序 | 展示商品图片、名称、规格选择器、所需积分、剩余库存 |
| 3 | 商品创建/编辑 | PC | 填写商品信息、管理规格组与 SKU（统一积分、主规格组规格图、库存），需指定所属积分类型 |
| 4 | 上架/下架/仓库 | PC | 商品状态流转管理 |
| 5 | 分类管理 | PC | 商品分类的增删改 |
| 6 | 库存管理 | PC | SKU 级库存管理 + 商品级限购规则 |

## 业务规则

### 商品池隔离

- 大RF商品和小RF商品各自独立，通过积分类型标签区分，互不混用
- 小程序端为同一页面顶部 Tab 切换（RF超市 / rf幼崽超市），未绑定小朋友的用户隐藏小RF Tab

### 商品状态与流转

商品有三种状态，对应枚举：`DRAFT`（草稿）、`WAREHOUSE`（仓库）、`ON_SALE`（已上架）。

**状态说明**

| 状态 | 枚举值 | 说明 |
|------|--------|------|
| 草稿 | DRAFT | 新建未完成，仅后台可见，不可在小程序展示 |
| 仓库 | WAREHOUSE | 编辑完成待上架 / 已下架，仅后台可见，可批量上架 |
| 已上架 | ON_SALE | 在小程序可见、可兑换 |

**允许的流转**

```
草稿 ──保存并放入仓库──→ 仓库
仓库 ──上架──→ 已上架
仓库 ──编辑后仍为仓库──→ 仓库（不变）
已上架 ──下架──→ 仓库
```

- 草稿 → 仓库：保存并「放入仓库」后生效；仓库/已上架均可编辑信息，不改变状态需单独操作。
- 仅「已上架」商品在小程序列表与详情中展示；下架后回到仓库状态，不再展示，但订单与库存数据保留。

### 商品与分类删除

- **商品删除**：任何状态的商品都可以删除（软删除）。删除时会同步清理该商品下的 SKU、购物车中关联该商品的条目、以及规格关联数据。已有的历史订单不受影响（订单明细通过 `productUuid` 冗余字段保留关联）。
- **分类删除**：仅允许删除**没有关联商品**的分类。若分类下仍存在商品（含任何状态），则拒绝删除并提示"该分类下仍有关联商品，暂不能删除"。

### 库存与限购

- 库存按 **SKU** 维度管理，后台可自动计算每个用户在各 SKU 上的可兑换数量
- 限购按 **商品（SPU）** 维度，所有 SKU 合计计算：如限购 N 件/月，用户当月兑换该商品下任意 SKU 总计不超过 N
- 单个 SKU 库存为 0 时显示「已兑完」；商品下所有 SKU 库存均为 0 时，商品整体显示「已兑完」

### 规格与 SKU

- 商品支持 **多规格**，每个商品可定义 0~N 组规格（如「颜色」「规格」「时长」等），规格组数量无上限
- 每组规格包含若干规格值（如颜色 → 红/蓝/白）
- 一个商品可将其中 **1 个规格组** 设置为 **主规格组**，主规格组主要用于承载规格图片
- 主规格组下，不同规格值可分别上传对应图片（如颜色组中的红色图、蓝色图）；非主规格组不配置规格图片
- 各规格值的笛卡尔积自动生成 **SKU**（库存单元），每个 SKU 仅维护库存，不再单独维护积分与图片
- 商品的兑换积分在 **商品（SPU）** 维度统一维护，同一商品下所有 SKU 所需积分一致
- 若商品无需规格（如停车券），则系统自动生成一个默认 SKU，后台仅需维护商品统一积分与默认 SKU 库存
- 小程序商品详情页展示规格选择器，用户选定规格后显示对应 SKU 库存；若所选主规格值配置了图片，则同步切换展示图
- 商品列表/卡片上直接展示商品统一积分，不再展示「xx 积分起」

### 商品类型

上架商品包括但不限于：
- 球场使用时间、课程
- 实物商品（水、手胶等）
- 停车券、代金券（券类商品，`Product.isCoupon = true`）

### 券类商品

券是一种特殊商品，通过 `Product.isCoupon = true` 标记，兑换后额外生成 `UserCoupon` 券实例。

**与普通商品的差别**：

| | 普通商品 | 券类商品 |
|---|---|---|
| 标记 | `isCoupon = false` | `isCoupon = true` |
| 兑换后 | 生成订单明细，走核销码核销 | 同左 + 额外生成 `UserCoupon` 记录 |
| 附加模板 | 无 | 关联 `CouponTemplate`（面额/折扣、有效期、使用规则） |
| 券包展示 | 无 | 用户在"我的券包"查看持有的券 |
| 券核销 | 无 | 工作人员在后台手动点击"已核销"，状态改为 USED |

**券的生命周期**：

```
后台创建商品（isCoupon=true）并关联券模板 → 上架 → 用户用积分兑换 → 生成 UserCoupon（UNUSED）→ 线下使用 → 工作人员手动核销（USED）
```

**券状态**（`UserCouponStatus`）：

| 状态 | 枚举值 | 说明 |
|------|--------|------|
| 未使用 | UNUSED | 已兑换但未使用，展示核销码 |
| 已使用 | USED | 工作人员已核销 |
| 已过期 | EXPIRED | 超过有效期未使用（定时任务自动标记） |

**券的使用限制**：

- 线上不做任何使用限制（无门槛、可叠加、不限总量），具体使用规则由线下工作人员把控
- 券过期后不退回积分
- 大RF和小RF积分均可兑换券（按商品所属积分类型决定）

**券相关数据模型**：

CouponTemplate（券模板）：

| 字段 | 类型 | 说明 |
|------|------|------|
| uuid | String | 主键 |
| productUuid | String? | 对应的券类商品 |
| name | String? | 券名称 |
| type | CouponType | FIXED_AMOUNT（代金券）/ PERCENT（折扣券） |
| amountOrPercent | Decimal? | 面额或折扣比例 |
| validFrom | DateTime | 生效时间 |
| validTo | DateTime | 失效时间 |
| rules | String? | 使用规则说明 |

UserCoupon（用户持有的券）：

| 字段 | 类型 | 说明 |
|------|------|------|
| uuid | String | 主键 |
| memberUuid | String | 持券会员 |
| couponTemplateUuid | String | 券模板 |
| orderItemUuid | String? | 对应的兑换订单明细 |
| status | UserCouponStatus | UNUSED / USED / EXPIRED |
| verifyCode | String? | 券核销码 |
| expiresAt | DateTime? | 到期时间（优先；为空回退到模板 validTo） |
| usedAt | DateTime? | 实际使用时间 |

**券相关接口**：

| 接口 | 路径 | 端 | 说明 |
|------|------|---|------|
| 我的券包 | `POST /coupons/my` | 小程序 | 按状态筛选（UNUSED/USED/EXPIRED） |
| 券详情 | `POST /coupons/detail` | 小程序 | 面额、规则、有效期、核销码 |
| 券模板列表 | `POST /admin/coupon-templates/list` | PC | 查看所有券模板 |
| 券核销 | `POST /admin/coupons/verify` | PC | 核销码核销，状态改为 USED |

**券相关定时任务**（已在 `POST /cron/daily` 中实现）：

- 过期券状态更新：`UNUSED` 且已过期 → 自动改为 `EXPIRED`
- 券即将过期提醒：到期前 3 天写消息 `COUPON_EXPIRY_WARN`，同一张券只发一次

### 商品展示规则

- 有简单筛选和积分排序功能
- 优先展示用户能购买的商品
- 展示顺序：按排序权重降序，同权重按上架时间降序，支持积分从低到高/从高到低手动切换排序

## 数据字段

### 商品（SPU）

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 商品名称 | 文本 | 是 | 商品标题 |
| 商品图片 | 图片 | 是 | 多张，JSON 数组存储 URL，**首张为封面**；列表、详情均可用 |
| 所需积分 | 数字 | 是 | 商品统一兑换积分，同一商品下所有 SKU 一致 |
| 积分类型 | 枚举 | 是 | 大RF(BIG_RF) / 小RF(SMALL_RF)，创建后不可更改 |
| 商品分类 | 引用 | 是 | 所属分类（大RF和小RF共用一套分类） |
| 限购规则 | 数字 | 否 | 每人每月限兑数量（所有 SKU 合计），0=不限 |
| 商品详情 | 富文本 | 否 | 详细介绍，支持两种格式：HTML（`description`）和可视化装修块（`descriptionBlocks`） |
| 商品装修块 | JSON | 否 | 可视化编辑的内容块数组（文字块/图片块），支持字号、颜色、对齐，优先于 HTML 渲染 |
| 是否券类 | 布尔 | 否 | `isCoupon` 标记，券类商品额外生成 UserCoupon（详见券类商品章节） |
| 状态 | 枚举 | 是 | 草稿(DRAFT)/仓库(WAREHOUSE)/已上架(ON_SALE) |
| 排序权重 | 数字 | 否 | 数字越大越靠前，同权重按上架时间排序 |

**图片约定**：商品表存多图 URL 列表（如 `["url1","url2"]`），首张即主图/封面；列表与详情默认用商品图。若商品设置了主规格组，且当前选中的主规格值配置了规格图片，则商品详情页优先展示该规格图片。

### 分类

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 分类名称 | 文本 | 是 | 如「饮料」「课程」 |
| 排序 | 数字 | 否 | 分类在筛选/导航中的展示顺序，数值越大越靠前 |

- 分类全局一套，大RF、小RF商品共用；通过商品的「积分类型」区分所属积分池。

### 规格体系说明（与表结构对应）

规格分为两层：

1. **全局规格类别（规格名称 + 规格值库）**  
   如「颜色」下列出「红色、蓝色、白色」，「尺码」下列出「S、M、L」。  
   对应表：`ProductSpecCategory`（规格名称、排序）、`ProductSpecOption`（规格值、同类别内唯一、排序）。  
   多个商品可复用同一规格类别。

2. **商品使用的规格组**  
   每个商品可选用 0~N 个规格类别，并为每个类别勾选本商品要用的规格值。  
   对应表：`ProductSpec`（商品 + 规格类别 + 顺序）、`ProductSpecValue`（该规格下选中的 option）。  
   前端编辑时表现为：为商品添加「规格组」→ 选规格名称（或新建）→ 勾选规格值列表并排序。

### 规格组（商品维度）

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 规格名称 | 文本 | 是 | 如「颜色」「规格」「时长」，来自全局规格类别 |
| 规格值列表 | 选项[] | 是 | 从该规格类别下勾选的若干规格值，如 [红色, 蓝色, 白色] |
| 是否主规格组 | 布尔 | 否 | 一个商品最多仅能设置 1 个主规格组；主规格组用于规格图片切换 |
| 排序 | 数字 | 否 | 规格组在商品详情页中的展示顺序 |

> 一个商品可有 0~N 个规格组。无规格组时系统自动生成唯一默认 SKU。若存在多个规格组，最多仅允许 1 个规格组被标记为主规格组。

### 主规格组规格值图片

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 所属规格组 | 引用 | 是 | 必须是当前商品已设置且被标记为主规格组的规格组 |
| 规格值 | 引用 | 是 | 主规格组中的某个规格值，如「红色」「蓝色」 |
| 规格图片 | 图片 | 否 | 选中该规格值时用于商品详情主图展示；未设置则沿用商品默认图片 |

### SKU

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 规格组合 | 文本 | 自动 | 由所选规格值拼接而成，如「红色 / L」，用于展示与对账 |
| 库存数量 | 数字 | 是 | 该 SKU 可兑换总数 |
| SKU 图片 | 图片 | 否 | SKU 专属图片，可覆盖商品主图展示（如单独拍摄的实物图） |

## 表结构与文档对应关系

建议的表结构与上述需求、字段的对应关系如下，可支撑文档中的状态、图片、分类、规格与 SKU 规则。

| 文档概念 | 表/字段 | 说明 |
|----------|---------|------|
| 商品（SPU） | `Product` | 名称、统一积分、积分类型、分类、限购、详情、状态、排序权重 |
| 商品图片 | `Product.images` | Text 存 JSON 数组 URL，首张即封面/主图 |
| 商品状态 | `Product.status` | 枚举 DRAFT / WAREHOUSE / ON_SALE，与文档三种状态一致 |
| 分类 | `ProductCategory` | 名称、排序；大/小 RF 共用，商品通过 `categoryUuid` 关联 |
| 规格名称 + 规格值库 | `ProductSpecCategory` + `ProductSpecOption` | 全局复用，如「颜色」下挂「红/蓝/白」 |
| 商品规格组 | `ProductSpec` + `ProductSpecValue` | 商品引用规格类别并勾选具体 option，支持多组、排序，并可标记主规格组 |
| 主规格组规格图片 | `ProductSpecValue.image` | 仅主规格组下的规格值可配置图片，详情页按所选规格值切图 |
| 商品装修块 | `Product.descriptionBlocks` | JSON 格式内容块（文字块/图片块），PC 端有独立装修编辑器 |
| 券类标记 | `Product.isCoupon` | 区分普通商品与券类商品 |
| SKU | `ProductSku` | `specSnapshot` 规格组合文案、`stock`、`image`（积分统一用 SPU 级 `pointsPrice`） |
| 库存流水 | `StockRecord` | 按 SKU 记录入库/出库，便于对账与审计 |

**设计要点简述**

- **规格复用**：规格类别与选项全局维护，商品只做「引用 + 选值」，避免重复维护「颜色」「尺码」等名称与取值。
- **状态单一字段**：用一枚 `status` 枚举覆盖草稿→仓库→上架，下架后回到仓库，流转由业务逻辑控制，无需多表。
- **积分统一**：积分放在商品（SPU）维度统一维护，SKU 仅承载库存和图片，避免同商品下出现多套积分。
- **图片分层**：SPU 级多图用于列表/详情轮播，主规格组规格值图片用于详情页按规格切图，避免每个 SKU 都单独传图。
- **分类与积分类型**：分类不区分大/小 RF，商品用 `pointsType` 区分积分池，列表与筛选按积分类型 + 分类组合即可。
