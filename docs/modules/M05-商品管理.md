# 模块05：商品管理

**职责**：积分商城中可兑换商品的全生命周期管理

## 能力清单

| # | 能力 | 端 | 描述 |
|---|------|---|------|
| 1 | 商品浏览 | 小程序 | 按积分类型（大RF/小RF）分别展示商品列表，支持分类筛选、积分排序 |
| 2 | 商品详情 | 小程序 | 展示商品图片、名称、规格选择器、所需积分、剩余库存 |
| 3 | 商品创建/编辑 | PC | 填写商品信息、管理规格组与 SKU（积分价格、库存），需指定所属积分类型 |
| 4 | 上架/下架/仓库 | PC | 商品状态流转管理，支持批量上架/批量下架 |
| 5 | 分类管理 | PC | 商品分类的增删改 |
| 6 | 库存管理 | PC | SKU 级库存管理 + 商品级限购规则 |

## 业务规则

### 商品池隔离

- 大RF商品和小RF商品各自独立，通过积分类型标签区分，互不混用
- 小程序端展示方式待确认（分开展示 or 混合展示+标签筛选，见待澄清问题）

### 商品状态与流转

商品有四种状态，对应枚举：`DRAFT`（草稿）、`WAREHOUSE`（仓库）、`ON_SALE`（已上架）、`OFF_SALE`（已下架）。

**状态说明**

| 状态 | 枚举值 | 说明 |
|------|--------|------|
| 草稿 | DRAFT | 新建未完成，仅后台可见，不可在小程序展示 |
| 仓库 | WAREHOUSE | 编辑完成待上架，仅后台可见，可批量上架 |
| 已上架 | ON_SALE | 在小程序可见、可兑换 |
| 已下架 | OFF_SALE | 不再展示与兑换，历史订单保留 |

**允许的流转**

```
草稿 ──保存并放入仓库──→ 仓库
仓库 ──上架──→ 已上架
仓库 ──编辑后仍为仓库──→ 仓库（不变）
已上架 ──下架──→ 已下架
已下架 ──上架──→ 已上架
```

- 草稿 → 仓库：保存并「放入仓库」后生效；仓库/已上架/已下架均可编辑信息，不改变状态需单独操作。
- 仅「已上架」商品在小程序列表与详情中展示；下架后不再展示，但订单与库存数据保留。

### 库存与限购

- 库存按 **SKU** 维度管理，后台可自动计算每个用户在各 SKU 上的可兑换数量
- 限购按 **商品（SPU）** 维度，所有 SKU 合计计算：如限购 N 件/月，用户当月兑换该商品下任意 SKU 总计不超过 N
- 单个 SKU 库存为 0 时显示「已兑完」；商品下所有 SKU 库存均为 0 时，商品整体显示「已兑完」

### 规格与 SKU

- 商品支持 **多规格**，每个商品可定义 0~N 组规格（如「颜色」「规格」「时长」等），规格组数量无上限
- 每组规格包含若干规格值（如颜色 → 红/蓝/白）
- 各规格值的笛卡尔积自动生成 **SKU**（库存单元），每个 SKU 独立维护积分价格与库存
- 若商品无需规格（如停车券），则系统自动生成一个默认 SKU，后台表现为直接填写积分和库存，无需额外操作
- 小程序商品详情页展示规格选择器，用户选定规格后显示对应 SKU 的积分与库存
- 商品列表/卡片上展示该商品所有 SKU 中的最低积分，格式如「200 积分起」

### 商品类型

来自会议纪要，上架商品包括但不限于：
- 球场使用时间、课程
- 实物商品（水、手胶等）
- 停车券、代金券（券类作为特殊商品，见优惠券模块）

### 商品展示规则

- 有简单筛选和积分排序功能
- 优先展示用户能购买的商品
- 展示顺序：按排序权重降序，同权重按上架时间降序，支持积分从低到高/从高到低手动切换排序

## 数据字段

### 商品（SPU）

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 商品名称 | 文本 | 是 | 商品标题 |
| 商品图片 | 图片 | 是 | 多张，JSON 数组存储 URL，**首张为封面**；列表、详情均可用 |
| 积分类型 | 枚举 | 是 | 大RF(BIG_RF) / 小RF(SMALL_RF)，创建后不可更改 |
| 商品分类 | 引用 | 是 | 所属分类（大RF和小RF共用一套分类） |
| 限购规则 | 数字 | 否 | 每人每月限兑数量（所有 SKU 合计），0=不限 |
| 商品详情 | 富文本 | 否 | 详细介绍（HTML） |
| 状态 | 枚举 | 是 | 草稿(DRAFT)/仓库(WAREHOUSE)/已上架(ON_SALE)/已下架(OFF_SALE) |
| 排序权重 | 数字 | 否 | 数字越大越靠前，同权重按上架时间排序 |

**图片约定**：商品表存多图 URL 列表（如 `["url1","url2"]`），首张即主图/封面；列表与详情默认用主图，若某 SKU 设置了专属图则在该 SKU 下用 SKU 图。

### 分类

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 分类名称 | 文本 | 是 | 如「饮料」「课程」 |
| 排序 | 数字 | 否 | 分类在筛选/导航中的展示顺序，数值越大越靠前 |

- 分类全局一套，大RF、小RF商品共用；通过商品的「积分类型」区分所属积分池。

### 规格体系说明（与表结构对应）

规格分为两层：

1. **全局规格类别（规格名称 + 规格值库）**  
   如「颜色」下列出「红色、蓝色、白色」，「尺码」下列出「S、M、L」。  
   对应表：`ProductSpecCategory`（规格名称、排序）、`ProductSpecOption`（规格值、同类别内唯一、排序）。  
   多个商品可复用同一规格类别。

2. **商品使用的规格组**  
   每个商品可选用 0~N 个规格类别，并为每个类别勾选本商品要用的规格值。  
   对应表：`ProductSpec`（商品 + 规格类别 + 顺序）、`ProductSpecValue`（该规格下选中的 option）。  
   前端编辑时表现为：为商品添加「规格组」→ 选规格名称（或新建）→ 勾选规格值列表并排序。

### 规格组（商品维度）

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 规格名称 | 文本 | 是 | 如「颜色」「规格」「时长」，来自全局规格类别 |
| 规格值列表 | 选项[] | 是 | 从该规格类别下勾选的若干规格值，如 [红色, 蓝色, 白色] |
| 排序 | 数字 | 否 | 规格组在商品详情页中的展示顺序 |

> 一个商品可有 0~N 个规格组。无规格组时系统自动生成唯一默认 SKU。

### SKU

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 规格组合 | 文本 | 自动 | 由所选规格值拼接而成，如「红色 / L」，用于展示与对账 |
| 所需积分 | 数字 | 是 | 该 SKU 兑换所需积分数 |
| 库存数量 | 数字 | 是 | 该 SKU 可兑换总数 |
| SKU 图片 | 图片 | 否 | 按规格设置不同图片（如不同颜色展示不同图）；**未设置时沿用商品主图（首图）** |

## 表结构与文档对应关系

当前实现与上述需求、字段的对应关系如下，表设计可支撑文档中的状态、图片、分类、规格与 SKU 规则。

| 文档概念 | 表/字段 | 说明 |
|----------|---------|------|
| 商品（SPU） | `Product` | 名称、积分类型、分类、限购、详情、状态、排序权重 |
| 商品图片 | `Product.images` | Text 存 JSON 数组 URL，首张即封面/主图 |
| 商品状态 | `Product.status` | 枚举 DRAFT / WAREHOUSE / ON_SALE / OFF_SALE，与文档四种状态一致 |
| 分类 | `ProductCategory` | 名称、排序；大/小 RF 共用，商品通过 `categoryUuid` 关联 |
| 规格名称 + 规格值库 | `ProductSpecCategory` + `ProductSpecOption` | 全局复用，如「颜色」下挂「红/蓝/白」 |
| 商品规格组 | `ProductSpec` + `ProductSpecValue` | 商品引用规格类别并勾选具体 option，支持多组、排序 |
| SKU | `ProductSku` | `specSnapshot` 规格组合文案，`pointsPrice`、`stock`、`image`（可选，未设用商品主图） |
| 库存流水 | `StockRecord` | 按 SKU 记录入库/出库，便于对账与审计 |

**设计要点简述**

- **规格复用**：规格类别与选项全局维护，商品只做「引用 + 选值」，避免重复维护「颜色」「尺码」等名称与取值。
- **状态单一字段**：用一枚 `status` 枚举覆盖草稿→仓库→上架→下架，流转由业务逻辑控制，无需多表。
- **图片分层**：SPU 级多图（列表/详情轮播）+ SKU 级单图可选（如按颜色换图），与文档约定一致。
- **分类与积分类型**：分类不区分大/小 RF，商品用 `pointsType` 区分积分池，列表与筛选按积分类型 + 分类组合即可。
