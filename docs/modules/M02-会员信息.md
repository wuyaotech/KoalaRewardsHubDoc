# 模块02：会员信息

**职责**：管理会员的基本资料和亲子关系

## 能力清单

| # | 能力 | 端 | 描述 |
|---|------|---|------|
| 1 | 个人信息填写/编辑 | 小程序 | 用户自行填写姓名、手机号、生日、性别等信息 |
| 2 | 小朋友管理 | 小程序 | 添加/编辑/删除小朋友信息（支持多个），添加后可查看小RF积分 |
| 3 | 会员列表查询 | PC | 工作人员搜索、筛选、查看所有会员，支持导出 |
| 4 | 会员信息修改 | PC | 工作人员手动修正用户录入的错误信息 |
| 5 | 会员详情 | PC | 查看单个会员的完整画像（资料+积分+等级+兑换+券） |

---

## 业务规则

1. **信息来源**：基本信息以用户主动授权填写为主，后台可手动更改
2. **小朋友是会员的子属性**：一个会员可以添加多个小朋友，每个小朋友有独立的信息和独立的小RF积分账户
3. **添加小朋友后**：该会员账号下会出现小RF积分区域，按小朋友分别展示积分
4. **手机号修改**：用户端不可直接修改手机号，需联系工作人员在后台操作

---

## 数据字段

### 会员基本信息

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 头像 | 图片 | 否 | 微信头像或自定义 |
| 姓名 | 文本 | 是 | 真实姓名 |
| 手机号 | 文本 | 是 | 绑定手机号，业务唯一标识 |
| 性别 | 枚举 | 否 | 男/女 |
| 生日 | 日期 | 否 | 年月日 |

### 小朋友信息（会员的子属性，支持多个）

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 小朋友姓名 | 文本 | 是 | 小朋友真实姓名 |
| 性别 | 枚举 | 否 | 男/女 |
| 生日 | 日期 | 否 | 年月日 |

> 每个小朋友拥有独立的小RF积分账户，录入积分时需选择具体哪个小朋友。
