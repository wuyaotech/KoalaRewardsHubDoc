# 模块02：会员信息

**职责**：管理会员的基本资料和亲子关系

## 能力清单

| # | 能力 | 端 | 描述 |
|---|------|---|------|
| 1 | 线下开卡/会员建档 | PC | 工作人员为到店会员录入手机号、姓名等信息，先建立会员档案 |
| 2 | 个人信息填写/编辑 | 小程序 | 用户自行补充或修改姓名、生日、性别等信息 |
| 3 | 小朋友管理 | 小程序 | 添加/编辑/删除小朋友信息（支持多个），添加后可查看小RF积分 |
| 4 | 会员列表查询 | PC | 工作人员搜索、筛选、查看所有会员，支持导出 |
| 5 | 会员信息修改 | PC | 工作人员手动修正建档或用户录入的错误信息 |
| 6 | 会员详情 | PC | 查看单个会员的完整画像（资料+积分+等级+兑换+券） |

---

## 业务规则

1. **会员档案可先于登录存在**：工作人员可在线下办卡/充值时先建立会员档案，无需等待用户先登录小程序
2. **首次登录自动匹配**：用户首次在小程序绑定手机号时，若系统中已有同手机号会员档案，则自动绑定并激活
3. **注册即入会**：用户首次绑定手机号时，若系统中不存在对应档案，则自动创建会员档案（来源为"注册转会员"）并初始化大RF积分账户
4. **手机号唯一**：一个手机号只能对应一个会员档案；若手机号已被其他登录账号绑定，则不允许重复绑定
5. **入会时机**：用户绑定手机号时自动入会，或由工作人员在线下办卡/建档时创建会员档案
6. **信息来源**：线下建档信息由工作人员录入，线上资料由用户主动补充；后台可手动修正
7. **小朋友是会员的子属性**：一个会员可以添加多个小朋友，每个小朋友有独立的信息和独立的小RF积分账户
8. **添加小朋友后**：该会员账号下会出现小RF积分区域，按小朋友分别展示积分
9. **手机号修改**：用户端不可直接修改手机号，需联系工作人员在后台操作，并保留修改记录

---

## 推荐流程

1. 会员在线下门店办卡、充值或首次到店消费
2. 工作人员在 PC 后台录入手机号、姓名等基础信息，创建一条会员档案
3. 会员档案先进入"未激活"状态，但可以承载积分、等级、到期时间等业务数据
4. 用户后续首次打开小程序，完成微信授权并绑定手机号
5. 系统按手机号匹配已有会员档案
6. 若匹配成功，则绑定登录账号并将会员状态更新为"已激活"
7. 若未匹配成功，则自动创建新的会员档案（来源标记为"注册转会员"），并初始化大RF积分账户

### 用户类型划分

| 类型 | 是否可登录小程序 | 是否属于正式会员 | 说明 |
|------|:----------------:|:----------------:|------|
| 未绑定手机号用户 | ✓ | ✗ | 仅完成了微信授权登录，尚未绑定手机号 |
| 未激活会员 | ✗ / 首次登录前 | ✓ | 已在线下建档，尚未完成小程序绑定 |
| 已激活会员 | ✓ | ✓ | 绑定手机号后自动成为会员，或线下建档后完成小程序绑定 |

## 数据字段

### 会员基本信息

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 会员状态 | 枚举 | 是 | 未激活 / 已激活 / 已停用 |
| 建档来源 | 枚举 | 是 | 线下录入 / 注册转会员 / 后台补建 |
| 头像 | 图片 | 否 | 微信头像或自定义 |
| 姓名 | 文本 | 是 | 真实姓名 |
| 手机号 | 文本 | 是 | 绑定手机号，业务唯一标识 |
| 性别 | 枚举 | 否 | 男/女 |
| 生日 | 日期 | 否 | 年月日 |
| 绑定时间 | 日期时间 | 否 | 首次完成账号绑定的时间 |
| 绑定账号标识 | 文本 | 否 | 关联的小程序登录账号标识 |

> 说明：用户绑定手机号后即自动创建会员档案，不再存在"已注册未入会"状态。仅微信授权但未绑定手机号的用户尚无会员档案。

### 小朋友信息（会员的子属性，支持多个）

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 小朋友姓名 | 文本 | 是 | 小朋友真实姓名 |
| 性别 | 枚举 | 否 | 男/女 |
| 生日 | 日期 | 否 | 年月日 |

> 每个小朋友拥有独立的小RF积分账户，录入积分时需选择具体哪个小朋友。
