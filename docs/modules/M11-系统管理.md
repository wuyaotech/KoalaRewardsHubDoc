# 模块11：系统管理

**职责**：后台自身的运维管理（仅PC后台）

## 能力清单

| # | 能力 | 端 | 描述 |
|---|------|---|------|
| 1 | 用户列表管理 | PC | 查看所有注册用户，设置/取消管理员角色 |
| 2 | 操作日志 | PC | 积分录入/修改、核销等关键操作的审计记录 |
| 3 | 积分规则配置 | PC | 积分比例、有效期规则等基础配置 |

---

## 业务规则

### 角色体系（三级）

| 角色 | 权限范围 | 来源 |
|------|----------|------|
| 超管 | 全部权限 + 设置/取消管理员 | 数据库写死，部署时初始化，不可通过界面创建 |
| 管理员 | 日常管理操作（积分录入、核销、商品管理、数据查看等） | 超管从用户列表中设置 |
| 普通用户 | 仅小程序端使用，无PC后台权限 | 小程序注册即为普通用户 |

### 超管专属操作

- 从用户列表中将普通用户设置为管理员（设置初始密码）
- 取消管理员权限（降为普通用户，不可再登录PC后台）
- 积分规则配置等系统级设置

### 管理员操作

- 积分录入/更正
- 核销操作
- 商品管理（上架/下架/编辑）
- 优惠券管理
- 查看兑换记录、数据统计
- 会员信息查看与修改

### 不单独建工作人员账号

所有账号统一来自小程序注册的用户池，通过角色标记区分权限，不需要在PC端独立创建账号。

### 操作日志

需要记录的关键操作：

| 操作类型 | 说明 |
|----------|------|
| 积分录入 | 谁、给谁、录了多少、什么时候 |
| 积分更正 | 谁、改了谁的、改了多少、原因 |
| 核销操作 | 谁、核销了哪个订单 |
| 商品上下架 | 谁、操作了什么商品 |
| 账号管理 | 谁、创建/禁用了哪个账号 |

- 操作日志永久保留，不做定期清理

### 积分规则配置

| 配置项 | 说明 |
|--------|------|
| 大RF积分比例 | 消费金额与积分的换算比例（默认 1元=1积分） |
| 大RF积分有效期 | 有效期规则 |
| 小RF积分有效期 | 小RF积分的有效期规则 |
| 清零提醒时间 | 清零前多久开始提醒 |

- 规则修改仅对后续新数据生效，不追溯历史数据
