# 模块01：用户与认证

**职责**：处理用户的身份识别和登录态管理

## 能力清单

| # | 能力 | 端 | 描述 |
|---|------|---|------|
| 1 | 微信授权登录 | 小程序 | 用户打开小程序，通过微信授权一键登录，获取 openid |
| 2 | 手机号绑定与自动注册会员 | 小程序 | 首次登录后绑定手机号；若系统中已有线下会员档案则自动匹配并激活，否则自动创建会员档案 |
| 3 | 会员档案激活 | 小程序/PC | 线下已建档会员首次登录后，完成账号绑定，成为可登录会员 |
| 4 | PC后台登录（密码） | PC | 管理员/超管使用手机号+密码登录后台管理系统 |
| 5 | PC后台登录（验证码） | PC | 管理员/超管使用手机号+短信验证码登录后台管理系统 |
| 6 | 管理员修改密码 | PC | 管理员/超管通过手机短信验证码自行修改登录密码 |
| 7 | 超管重置管理员密码 | PC | 超管可在用户管理页直接重置普通管理员的登录密码 |

## 会员注册模型

针对普通用户，系统采用"登录账号 + 会员档案"的双层模型，以适配线下先办卡/充值、线上先注册浏览两种业务场景。

### 模型说明

| 模型 | 创建时机 | 作用 |
|------|----------|------|
| 登录账号 | 用户实际进入小程序并完成登录/绑定时创建 | 承载登录身份，如微信 openid、登录态、最近登录时间等 |
| 会员档案 | 工作人员在 PC 后台录入，或用户绑定手机号时自动创建 | 承载会员业务信息，如姓名、手机号、会员状态、积分、等级、小朋友信息等 |

### 绑定关系

- 登录账号可独立存在（仅微信授权但未绑定手机号的阶段）
- 会员档案也可在未登录前独立存在，表示该用户已在线下建档但未激活
- 一个会员档案最多绑定一个登录账号
- 一个登录账号最多绑定一个会员档案
- 线下会员未登录前，状态为"未激活"
- 用户首次在小程序绑定手机号后，若匹配到已有会员档案，则直接绑定并激活
- 若未匹配到已有会员档案，则自动创建新的会员档案（来源标记为"注册转会员"），并同步创建大RF积分账户

### 角色定义

| 角色 | 来源 | 可登录小程序 | 可登录PC后台 | 说明 |
|------|------|:----------:|:----------:|------|
| 普通会员 | 小程序绑定手机号时自动注册，或线下建档后绑定 | ✓ | ✗ | 已进入会员体系，可承载积分、等级等业务 |
| 管理员 | 系统分配后台账号 | ✗ | ✓ | 可登录PC后台，执行日常管理操作 |
| 超管 | 数据库直接写入，不可通过界面创建 | ✗ | ✓ | 最高权限，可设置/取消管理员 |

### PC后台登录与密码管理

- 用户必须拥有后台账号
- PC 后台登录支持两种方式：手机号+密码 或 手机号+短信验证码
- 管理员初始密码由超管在设置管理员时指定
- 管理员/超管可在 PC 后台通过手机短信验证码自行修改密码（右上角用户菜单 → 修改密码）
- 超管可在用户管理页直接重置普通管理员的密码（操作会记录到操作日志）

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| uuid | String | 自动 | 主键 |
| memberProfileUuid | String | 否 | 关联的会员档案 UUID，绑定手机号后写入 |
| firstName | String | 否 | 名 |
| lastName | String | 否 | 姓 |
| phone | String | 否 | 手机号，登录标识，唯一 |
| passwordHash | String | 否 | 登录密码哈希（PC 后台管理员/超管） |
| avatar | String | 否 | 头像 URL（可用微信头像） |
| wechatOpenId | String | 否 | 小程序 openid，唯一 |
| wechatUnionId | String | 否 | 微信 unionid（多端打通），唯一 |
| status | 枚举 | 是 | ACTIVE（正常）/ INACTIVE（停用） |

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| uuid | String | 自动 | 主键 |
| phone | String | 是 | 接收验证码的手机号 |
| code | String | 是 | 验证码（4~6 位数字） |

## 业务规则

1. **会员业务主键**：手机号作为会员业务唯一标识，用于线下建档和线上绑定匹配
2. **登录标识**：微信 openid 作为小程序技术标识；PC端后台账号使用手机号+密码或手机号+验证码登录
3. **首次使用流程**：微信授权 → 手机号绑定 → 匹配已有会员档案；若匹配成功则激活会员，若未匹配则自动创建会员档案 → 进入首页
4. **绑定规则**：一个手机号只能绑定一个会员档案；若该手机号已绑定其他登录账号，直接提示冲突，不允许重复绑定
5. **线下优先**：工作人员可先在后台录入会员档案，用户后续首次登录时完成绑定
6. **注册即入会**：用户首次绑定手机号时，若系统中不存在对应会员档案，则自动创建会员档案并初始化大RF积分账户，来源标记为"注册转会员"
7. **PC后台登录**：支持手机号+密码和手机号+验证码两种方式，仅管理员和超管可登录，普通会员尝试登录提示无权限
8. **密码管理**：管理员/超管可通过手机验证码自行修改密码；超管可在用户管理页重置普通管理员密码
9. **超管不可通过界面创建**：超管账号直接写在数据库中，系统部署时初始化
