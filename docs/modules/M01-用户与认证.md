# 模块01：用户与认证

**职责**：处理用户的身份识别和登录态管理

## 能力清单

| # | 能力 | 端 | 描述 |
|---|------|---|------|
| 1 | 微信授权登录 | 小程序 | 用户打开小程序，通过微信授权一键登录，获取 openid |
| 2 | 手机号绑定 | 小程序 | 首次登录后绑定手机号，作为业务唯一标识；支持微信快捷获取手机号 |
| 3 | PC后台登录 | PC | 管理员/超管使用手机号+密码登录后台管理系统 |

---

## 统一账号体系

系统只有一个用户池，所有人通过小程序注册，手机号是唯一标识。角色通过权限标记区分，不单独建"工作人员账号"。

### 角色定义

| 角色 | 来源 | 可登录小程序 | 可登录PC后台 | 说明 |
|------|------|:----------:|:----------:|------|
| 普通用户 | 小程序注册即为普通用户 | ✓ | ✗ | 默认角色，只能使用小程序 |
| 管理员 | 超管在PC后台从用户列表中设置 | ✓ | ✓ | 可登录PC后台，执行日常管理操作 |
| 超管 | 数据库直接写入，不可通过界面创建 | ✓ | ✓ | 最高权限，可设置/取消管理员 |

### PC后台登录前提

- 用户必须先在小程序端完成注册（微信授权+手机号绑定）
- 超管在PC后台将该用户设置为管理员
- 管理员/超管使用手机号+密码登录PC后台
- 管理员初始密码为系统内置默认密码，首次登录后建议修改
- 管理员可通过手机验证码自行修改密码

---

## 业务规则

1. **用户标识**：微信 openid 作为技术标识，手机号作为业务唯一标识
2. **登录态**：小程序端登录态依赖微信 session，过期后需重新授权；PC端使用 token 机制
3. **首次使用流程**：微信授权 → 手机号绑定 → 进入首页（无需隐私协议勾选）
4. **账号唯一性**：一个手机号只能绑定一个账号，若手机号已被其他微信号绑定，直接提示"该手机号已被绑定"，不允许重复绑定
5. **PC后台登录**：手机号+密码，仅管理员和超管可登录，普通用户尝试登录提示无权限
6. **超管不可通过界面创建**：超管账号直接写在数据库中，系统部署时初始化
