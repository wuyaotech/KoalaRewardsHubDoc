# 模块06：兑换交易（含核销）

**职责**：用户从"想要"到"下单"再到"线下核销"的完整兑换闭环（大RF/小RF积分通用）

> 原 M07-核销 已合并至此模块。兑换下单后生成核销码、线下核销履约属于同一订单生命周期，代码实现中也由 `orders` + `verify` 两个模块协同完成，共享同一张 `RedeemOrder` 表。

## 能力清单

| # | 能力 | 端 | 描述 |
|---|------|---|------|
| 1 | 心愿单 | 小程序 | 收藏感兴趣的商品，类似收藏夹 |
| 2 | 购物车 | 小程序 | 积分足够的商品可加入购物车 |
| 3 | 兑换下单 | 小程序 | 确认兑换，扣减对应类型积分，生成订单与核销码 |
| 4 | 核销码展示 | 小程序 | 用户在小程序出示核销码（二维码 + 8位数字码）给前台 |
| 5 | 扫码/输码核销 | PC | 前台扫码或手动输入数字码完成核销 |
| 6 | 核销状态同步 | 双端 | 核销完成后用户端同步显示"已完成" |
| 7 | 兑换历史 | 小程序 | 查看个人兑换记录和状态（区分大RF/小RF） |
| 8 | 兑换记录查询 | PC | 按用户/积分类型/商品/时间段查看 |
| 9 | 核销历史 | PC | 按关键词查看已核销订单列表 |

## 业务规则

### 商品操作（心愿单 / 购物车）

- 单个商品支持两种操作，**无先后关系**：
  - **加入心愿单**：类似收藏夹，收藏感兴趣的商品
  - **加入购物车**：将商品加入购物车后统一结算
- 心愿单中商品下架后标记为"已下架"，不可操作

### 购物车

- 仅积分足够的商品可加入购物车
- 大RF商品和小RF商品在购物车中分组展示
- 购物车无数量上限
- 若会员绑定多个小朋友，小RF商品在加入购物车时需关联到具体小朋友，小RF积分仅能从该小朋友账户扣减

### 兑换下单

- 支持混合下单，大RF和小RF商品可同时结算，系统分别扣减对应类型的积分
- 小RF积分结算时，不同小朋友的小RF积分不可合并使用；每件小RF商品仅从其关联小朋友的小RF账户扣减
- 积分扣减不影响会员等级
- 兑换下单在事务内完成：创建订单 → 扣积分 → 扣库存 → 写积分流水 → 清空购物车中已结算项 → 生成核销码
- 兑换成功后自动生成核销码（8位数字码），有效期 **10 天**

### 核销流程

```
用户兑换成功 → 生成核销码 → 用户到前台出示 → 工作人员扫码/输码 → 确认核销 → 双端状态同步
```

- **核销终端**：前台使用 Pad/手机扫码优先，PC后台手动输码作为兜底
- **核销凭证**：每个兑换订单生成 1 个核销凭证（二维码 + 8位数字码）
- **不可撤销**：核销后状态不可逆，已核销如有问题走线下处理，由管理员手动重新发放积分

### 兑换订单状态

```
待核销(PENDING_VERIFY) → 已完成(VERIFIED)
待核销(PENDING_VERIFY) → 已取消(CANCELLED)
```

- **用户取消**：未核销的订单支持用户取消，取消后系统自动恢复库存并回退积分
- **超时取消**：超过 10 天未核销则订单自动取消（CANCELLED），恢复库存并回退积分

> 注意：代码中没有独立的"已过期"状态，过期与用户主动取消共用 `CANCELLED`。前端可根据 `verifyCodeExpiresAt` 在展示层区分"已过期"和"已取消"。

### 兑换记录

- 每个用户的兑换历史在后台都可见
- 也可以集中不按用户看，查看加总兑换数据，以便调节商品结构

## 数据模型（与 Prisma 对齐）

### RedeemOrder 兑换订单

| 字段 | 类型 | 说明 |
|------|------|------|
| uuid | String | 主键 |
| memberUuid | String | 下单会员 |
| childUuid | String? | 小RF积分对应的小朋友（仅含小RF商品时必填） |
| status | RedeemOrderStatus | PENDING_VERIFY / VERIFIED / CANCELLED |
| verifyCode | String | 核销数字码（8位） |
| verifyCodeExpiresAt | DateTime | 核销码过期时间（下单后 +10 天） |
| verifiedAt | DateTime? | 实际核销时间 |
| operatorUuid | String? | 核销操作人（后台管理员） |
| totalBigRfPoints | Int | 本单消耗大RF积分总额 |
| totalSmallRfPoints | Int | 本单消耗小RF积分总额 |

### RedeemOrderItem 订单明细

| 字段 | 类型 | 说明 |
|------|------|------|
| uuid | String | 主键 |
| orderUuid | String | 所属订单 |
| skuUuid | String | SKU |
| productUuid | String | 商品（冗余，便捷查询） |
| pointsType | PointsType | BIG_RF / SMALL_RF |
| pointsPrice | Int | 单件所需积分 |
| quantity | Int | 兑换数量 |
| pointsAmount | Int | 合计积分 = price × quantity |

## 接口清单（与代码对齐）

### 兑换（orders 模块）

| 接口 | 权限 | 说明 |
|------|------|------|
| `POST /orders/preview` | 登录用户 | 兑换预览：校验购物车商品、计算积分小计 |
| `POST /orders/create` | 登录用户 | 创建兑换订单（事务：扣积分+扣库存+生成核销码） |
| `POST /orders/list` | 登录用户 | 我的兑换记录 |
| `POST /admin/orders/list` | 管理员 | 后台兑换记录列表（分页、筛选） |

### 核销（verify 模块）

| 接口 | 权限 | 说明 |
|------|------|------|
| `POST /verify/orders/detail` | 登录用户 | 小程序查看订单核销信息（核销码、状态） |
| `POST /admin/verify/preview` | 管理员 | 扫码/输码后预览核销信息 |
| `POST /admin/verify/by-code` | 管理员 | 执行核销（更新状态、记录操作日志） |
| `POST /admin/verify/history` | 管理员 | 核销历史列表 |
