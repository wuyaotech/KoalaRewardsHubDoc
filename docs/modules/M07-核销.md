# 模块07：核销

**职责**：兑换后的线下履约闭环

## 能力清单

| # | 能力 | 端 | 描述 |
|---|------|---|------|
| 1 | 核销码生成 | 小程序 | 兑换成功后自动生成核销码（二维码 + 数字码） |
| 2 | 核销码展示 | 小程序 | 用户在小程序中出示核销码给前台工作人员 |
| 3 | 扫码核销 | PC | 前台用 Pad 或手机扫描用户核销码二维码 |
| 4 | 手动输码核销 | PC | 前台电脑上手动输入核销数字码完成核销 |
| 5 | 核销状态同步 | 双端 | 核销完成后用户端同步显示"已兑换" |

---

## 业务规则

1. **核销方式**：兑换实物产品后生成核销码，用户找工作人员领取后，工作人员在后台点击已核销
2. **双端同步**：工作人员核销后，用户小程序界面也应显示"已兑换"
3. **核销终端**：前台使用 Pad/手机核销页扫码优先，PC后台手动输码作为兜底
4. **核销凭证**：每个兑换订单生成 1 个核销凭证，包含二维码 + 8位数字码（扫码优先，输码兜底）
5. **有效期**：核销码有效期为 10 天，过期后不可核销
6. **不可撤销**：核销后状态不可撤销，不提供“撤销核销”能力

### 核销流程

```
用户兑换成功 → 生成核销码 → 用户到前台出示 → 工作人员扫码/输码 → 确认核销 → 双端状态同步
```

### 核销状态机

```
待核销（有效期内） → 已核销（完成）
待核销（超过7天） → 已过期（关闭）
```
