## 统计指标详情

### 一、工作台首页概览卡片（dashboard）

每次打开后台第一眼看到的数据，接口 `POST /admin/stats/dashboard` 返回：

| 字段 | 说明 | 计算方式 |
|------|------|----------|
| `totalMembers` | 总会员数 | `MemberProfile` 非软删总数 |
| `todayNew` | 今日新增会员 | 当日 `createdAt >= 今日0点` 的会员数 |
| `pendingActivationCount` | 待激活会员数 | `status = PENDING_ACTIVATION` 的会员数 |
| `pendingVerifyOrders` | 待核销订单数 | `status = PENDING_VERIFY` 的兑换订单数 |
| `stockAlertCount` | 库存预警商品数 | 在售 SKU 中 `stock ≤ 10` 的数量 |

> 注意：前端工作台页面目前展示了前 4 项，`stockAlertCount` 已在 API 返回但页面尚未渲染。

---

### 二、兑换统计（overview.exchange）

| 字段 | 说明 | 计算方式 |
|------|------|----------|
| `totalOrders` | 兑换总笔数 | `RedeemOrder` 非软删总数 |
| `verifiedOrders` | 已核销订单数 | `status = VERIFIED` 的订单数 |
| `totalPointsConsumed` | 兑换总积分消耗 | 所有订单 `totalBigRfPoints + totalSmallRfPoints` 之和 |
| `topProducts` | 热门商品 TOP 10 | 按 `RedeemOrderItem.quantity` 聚合排序，含商品名 / 积分类型 / 兑换次数 / 消耗积分 |

前端展示的衍生指标：
- **核销率** = `verifiedOrders / totalOrders`（前端计算）

---

### 三、会员统计（overview.members）

| 字段 | 说明 | 计算方式 |
|------|------|----------|
| `total` | 总会员数 | `MemberProfile` 非软删总数 |
| `todayNew` | 今日新增 | 当日创建的会员数 |
| `withChildren` | 有绑定小朋友的会员数 | 存在 `children` 关联的会员数 |
| `levelDistribution` | 等级分布 | 按 `levelUuid` 分组统计人数，关联 `MemberLevel.name` |

---

### 四、积分统计（overview.points）

按大RF / 小RF 分别统计，结构相同：

| 字段 | 说明 | 计算方式 |
|------|------|----------|
| `totalIssued` | 积分发放总量 | `PointsLedger` 中 `type = IN` 的 `amount` 合计 |
| `totalConsumed` | 积分消耗总量 | `PointsLedger` 中 `type = OUT` 的 `amount` 合计 |
| `currentBalance` | 当前流通积分总量 | `PointsAccount.balance` 合计 |

前端展示的衍生指标：
- **使用率** = `totalConsumed / totalIssued`（前端计算）

---

### 五、库存预警（overview.products）

| 字段 | 说明 | 计算方式 |
|------|------|----------|
| `stockAlerts` | 库存预警列表（前20条） | 在售 SKU 中 `0 < stock ≤ 10`，按库存升序，含商品名 / 库存 / 积分类型 / 规格 / 兑换次数 |
| `soldOutCount` | 已售罄 SKU 数 | 在售 SKU 中 `stock ≤ 0` 的数量 |

库存预警阈值：`STOCK_ALERT_THRESHOLD = 10`（代码硬编码）
