# 模块10：数据统计

**职责**：为运营决策提供数据支撑（仅PC后台）

## 能力清单

| # | 能力 | 端 | 描述 |
|---|------|---|------|
| 1 | 工作台概览 | PC | 首页关键指标卡片，快速掌握今日/整体状况 |
| 2 | 会员统计 | PC | 会员规模、等级分布、增长趋势 |
| 3 | 积分统计 | PC | 积分发放、消耗、清零、余量的全盘情况 |
| 4 | 兑换统计 | PC | 兑换量、热门商品、兑换趋势 |
| 5 | 库存预警 | PC | 库存即将耗尽的商品列表 |

---

## 统计指标详情

### 一、工作台首页概览卡片（实时/今日）

快速看当下状态，每次打开后台第一眼看到的数据：

| 指标 | 说明 | 用途 |
|------|------|------|
| 总会员数 | 累计注册会员总量 | 掌握用户规模 |
| 今日新增会员 | 当天新注册人数 | 感知用户增长 |
| 待核销订单数 | 当前未核销的兑换订单数 | 提醒工作人员处理 |
| 库存预警商品数 | 库存低于阈值的商品数量 | 提醒及时补货/下架 |

---

### 二、会员统计

**用途**：了解用户结构，判断等级权益是否有吸引力

| 指标 | 说明 |
|------|------|
| 总会员数 | 累计注册总量 |
| 各等级人数分布 | 考拉/小熊/小马/海豚/花豹各等级的人数和占比 |
| 有绑定小朋友的会员数 | 亲子用户规模 |
| 新增趋势图 | 按月/周的会员注册增长曲线 |

---

### 三、积分统计

**用途**：了解积分的健康度，判断积分是否在有效流通

| 指标 | 维度 | 说明 |
|------|------|------|
| 积分发放总量 | 大RF / 小RF | 工作人员累计录入的积分总数 |
| 积分消耗总量 | 大RF / 小RF | 兑换商品扣减的积分总数 |
| 当前流通积分总量 | 大RF / 小RF | 所有会员当前持有的积分余量之和 |
| 即将清零积分量 | 大RF / 小RF | 未来3个月内将清零的积分总量 |
| 已清零积分总量 | 大RF / 小RF | 历史清零的积分累计数 |
| 积分使用率 | 大RF / 小RF | 消耗量 / 发放量，反映积分活跃程度 |

---

### 四、兑换统计

**用途**：了解哪些商品受欢迎，调整商品结构

| 指标 | 维度 | 说明 |
|------|------|------|
| 兑换总笔数 | 大RF / 小RF / 时间段 | 整体兑换活跃度 |
| 兑换总积分消耗 | 大RF / 小RF / 时间段 | 积分的消耗速度 |
| 热门商品 TOP 10 | 大RF / 小RF | 按兑换次数排名，调节商品结构的核心依据 |
| 兑换趋势图 | 按月/周 | 兑换量随时间变化曲线 |
| 核销率 | — | 已核销订单 / 总兑换订单，反映履约情况 |

---

### 五、库存预警

**用途**：提醒工作人员关注库存紧张商品，避免超卖

| 指标 | 说明 |
|------|------|
| 库存低于N件的商品列表 | 展示商品名称、积分类型、当前库存、兑换量，N写死在数据库（默认10件），暂不做后台配置 |
| 已售罄商品数 | 库存为0但仍上架的商品，提醒及时处理 |
