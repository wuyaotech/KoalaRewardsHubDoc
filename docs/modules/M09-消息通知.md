# 模块09：消息通知

**职责**：应用内消息中心，通过小红点提示用户有未读消息

## 能力清单

| # | 能力 | 端 | 描述 |
|---|------|---|------|
| 1 | 消息红点提示 | 小程序 | 有未读消息时，消息中心入口显示红点 |
| 2 | 消息列表 | 小程序 | 查看所有消息，区分已读/未读 |
| 3 | 系统自动生成消息 | 系统 | 触发特定事件时自动写入消息 |

---

## 业务规则

### 通知方式

**仅应用内消息**，不使用微信服务通知/订阅消息/推送，用户打开小程序后看到红点，点进去查看即可。

### 消息状态

- 消息分为**未读**和**已读**
- 用户点击消息后标记为已读
- 红点在所有消息已读后消失

## 消息分类与优先级

按业务重要程度分为两档，用于列表中的视觉区分（如加粗/图标）：

| 优先级 | 消息类型 | 说明 |
|--------|----------|------|
| 重要 | `POINTS_EXPIRY_WARN`、`POINTS_EXPIRED`、`ORDER_CANCELLED_TIMEOUT`、`ORDER_PARTIAL_TIMEOUT`、`VERIFY_CODE_EXPIRING` | 涉及资产损失风险，需要用户关注 |
| 普通 | 其余所有类型 | 常规通知 |

## 触发方式汇总

消息的写入分两种触发方式：

### 实时触发（业务操作后立即写入）

| 触发点代码位置 | 写入的消息类型 |
|---------------|---------------|
| `PointsService.adminGrantBigPoints` | `POINTS_GRANTED_BIG` |
| `PointsService.adminGrantSmallPoints` | `POINTS_GRANTED_SMALL` |
| `PointsService.adminCorrectPoints` | `POINTS_CORRECTED` |
| `OrdersService.create` | `ORDER_CREATED`；若需单独礼券通知可再写 `COUPON_RECEIVED`（当前可未实现） |
| `VerifyService.verifyByCode` | `ORDER_VERIFIED` |
| 等级升级逻辑（录入积分/更正积分后） | `LEVEL_UPGRADED` |
| 会员激活逻辑（手机号绑定匹配后） | `MEMBER_ACTIVATED` |

### 定时任务触发（`POST /cron/daily` 内执行）

| 子任务 | 写入的消息类型 |
|--------|---------------|
| 积分清零提醒（距周年日 90/60/30 天） | `POINTS_EXPIRY_WARN` |
| 积分清零执行 | `POINTS_EXPIRED` |
| 超时订单取消（完全未核销） | `ORDER_CANCELLED_TIMEOUT` |
| 超时订单结算（部分核销） | `ORDER_PARTIAL_TIMEOUT` |
| 核销码即将过期（到期前 2 天，含券类商品订单） | `VERIFY_CODE_EXPIRING` |

## 点击跳转路径

| 跳转目标 | 小程序路径 | 需要参数 |
|----------|-----------|----------|
| 积分明细页（大RF） | `/pages/points-ledger/points-ledger?pointsType=BIG_RF` | pointsType |
| 积分明细页（小RF） | `/pages/points-ledger/points-ledger?pointsType=SMALL_RF&childUuid={childUuid}` | pointsType, childUuid |
| 积分商城页 | `/pages/mall/mall` | 无 |
| 核销码页 | `/pages/verify-code/verify-code?orderUuid={orderUuid}` | orderUuid |
| 兑换记录页 | `/pages/redeem-history/redeem-history` | 无 |
| 礼券列表 / 核销码 | `/pages/coupon-list/coupon-list` 或 `/pages/verify-code/verify-code?orderUuid={orderUuid}` | 视实现 |
| 个人中心页 | `/pages/profile/profile` | 无 |

> 消息表中存 `linkUrl` 字段（完整小程序页面路径含参数），点击消息时直接 `wx.navigateTo({ url: linkUrl })`；若 linkUrl 为空则点击仅标记已读、不跳转。

## 数据模型（与 Prisma 对齐）

```prisma
model Message {
  id        Int             @id @default(autoincrement())
  uuid      String          @unique @default(uuid()) @db.VarChar(36)
  userUuid  String          @db.VarChar(36)
  type      MessageType                        // 消息类型枚举
  priority  MessagePriority @default(NORMAL)   // IMPORTANT / NORMAL
  title     String          @db.VarChar(100)   // 消息标题
  content   String          @db.Text           // 消息正文
  linkUrl   String?         @db.VarChar(500)   // 点击跳转的小程序页面路径（含参数）
  isRead    Boolean         @default(false)
  bizUuid   String?         @db.VarChar(36)    // 关联的业务单据 uuid（订单/券/积分账户等），用于防重
  createdAt DateTime        @default(now())
  updatedAt DateTime        @updatedAt
  deletedAt DateTime?

  user User @relation(fields: [userUuid], references: [uuid])

  @@index([userUuid, isRead])
  @@index([userUuid, createdAt])
  @@index([type, bizUuid])
}

enum MessageType {
  POINTS_GRANTED_BIG
  POINTS_GRANTED_SMALL
  POINTS_CORRECTED
  POINTS_EXPIRY_WARN
  POINTS_EXPIRED
  ORDER_CREATED
  ORDER_VERIFIED
  ORDER_CANCELLED_TIMEOUT
  ORDER_PARTIAL_TIMEOUT
  VERIFY_CODE_EXPIRING
  COUPON_RECEIVED
  LEVEL_UPGRADED
  MEMBER_ACTIVATED
}

enum MessagePriority {
  IMPORTANT
  NORMAL
}
```

## 防重复规则

同一事件不应产生重复消息：

| 消息类型 | 防重策略 |
|----------|---------|
| `POINTS_EXPIRY_WARN` | 同一会员 + 同一清零周期 + 同一提醒档位（90/60/30天），检查是否已有同类型消息 |
| `VERIFY_CODE_EXPIRING` | 同一订单只发一次到期提醒 |
| 其他实时触发类型 | 跟随业务操作，天然不重复 |
