# 模块08：优惠券

**职责**：优惠券和代金券的创建、兑换、使用、核销

## 能力清单

| # | 能力 | 端 | 描述 |
|---|------|---|------|
| 1 | 我的券包 | 小程序 | 查看持有的券，区分状态（未使用/已使用/已过期） |
| 2 | 券详情 | 小程序 | 查看券面额、使用规则、有效期、核销码 |
| 3 | 创建券模板 | PC | 设置券类型、面额/折扣、有效期、使用规则 |
| 4 | 券核销 | PC | 工作人员手动点击"已核销" |

---

## 业务规则

1. **券的本质**：券在积分商城中作为一种特殊商品上架，用户用积分兑换获得
2. **券的使用**：兑换后在线下消费时使用（如采购付款时扣除），不与 POS 机绑定
3. **券的核销**：靠后台工作人员手动点击"已核销"，相当于商品一样手动核销
4. **券的类型**：优惠券（折扣）和代金券（固定金额），具体由后台设置

### 券的生命周期

```
后台创建券模板 → 作为商品上架 → 用户用积分兑换 → 持有在券包 → 线下使用 → 工作人员手动核销
```

### 券的使用限制

- 线上不做任何使用限制（无门槛、可叠加、不限总量），具体使用规则由线下工作人员把控
- 券过期后不退回积分
- 大RF和小RF积分均可兑换券（券作为普通商品上架，按商品所属积分类型决定）

### 券状态

| 状态 | 枚举值 | 说明 |
|------|--------|------|
| 未使用 | UNUSED | 用户已兑换但未使用，展示核销码 |
| 已使用 | USED | 工作人员已核销 |
| 已过期 | EXPIRED | 超过有效期未使用（定时任务自动标记） |

### 定时任务（详见 M13-定时任务）

- **过期券状态更新**：每天凌晨定时任务扫描 `status = UNUSED` 且已过期的券，自动更新为 `EXPIRED`
- **券即将过期提醒**：到期前 3 天发送消息通知 `COUPON_EXPIRY_WARN`，同一张券只发一次

## 数据模型

### CouponTemplate 券模板

| 字段 | 类型 | 说明 |
|------|------|------|
| uuid | String | 主键 |
| productUuid | String? | 对应的券类商品（`Product.isCoupon = true`） |
| name | String? | 券名称 |
| type | CouponType | FIXED_AMOUNT（代金券）/ PERCENT（折扣券） |
| amountOrPercent | Decimal? | 面额或折扣比例 |
| validFrom | DateTime | 生效时间 |
| validTo | DateTime | 失效时间（模板级统一到期时间） |
| rules | String? | 使用规则说明 |

### UserCoupon 用户持有的券

| 字段 | 类型 | 说明 |
|------|------|------|
| uuid | String | 主键 |
| memberUuid | String | 持券会员 |
| couponTemplateUuid | String | 券模板 |
| orderItemUuid | String? | 若由兑换得来，对应的订单明细 |
| status | UserCouponStatus | UNUSED / USED / EXPIRED |
| verifyCode | String? | 券核销码 |
| expiresAt | DateTime? | 券到期时间（创建时从模板 validTo 复制；未来可支持"兑换后 N 天有效"的个性化到期） |
| usedAt | DateTime? | 实际使用时间 |

> 到期判断优先取 `UserCoupon.expiresAt`，若为空则回退到 `CouponTemplate.validTo`。

## 接口清单（与代码对齐）

### 小程序端接口（需要登录）

| 接口 | 方法 | 路径 | 说明 |
|------|------|------|------|
| 我的券包 | POST | `/coupons/my` | 查看持有的券列表，可按状态筛选（UNUSED/USED/EXPIRED） |
| 券详情 | POST | `/coupons/detail` | 查看单张券的详细信息（面额、规则、有效期、核销码） |

### PC 后台接口（需要管理员权限）

| 接口 | 方法 | 路径 | 说明 |
|------|------|------|------|
| 券模板列表 | POST | `/admin/coupon-templates/list` | 查看所有券模板 |
| 券核销 | POST | `/admin/coupons/verify` | 工作人员通过核销码核销券，更新状态为 USED |

> 券模板的创建/编辑当前通过商品管理模块完成（券作为特殊商品，`Product.isCoupon = true`，关联 `CouponTemplate`）。
